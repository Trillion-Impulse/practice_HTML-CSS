# how to use HTML
***

## HTML

- HyperText Markup Language
- 웹 페이지의 구조와 내용을 정의하는 마크업 언어 
- 웹 페이지의 레이아웃과 콘텐츠를 표시하는 데 초점을 맞춘 마크업 언어
    - *마크업 언어: 문서의 구조와 형식을 정의하는 언어*
- 확장자로 .html 을 사용 (과거에는 .htm 도 사용)

## 태그 Tag

- 태그는 HTML 문서에서 콘텐츠의 종류와 구조를 정의하는 데 사용되는 기호
- < 와 > 로 감싸져 있으며 기호 사이에 태그명이 들어감
- 시작태그와 종료태그로 이루어짐

```
<h1>Hello, HTML</h1>
```

- 태그: `<h1>`, `</h1>`
- 태그명: `h1`
- 시작태그: `<h1>`
- 종료태그: `</h1>`
- 내용(Content): `Hello, HTML`
- 요소(Element): `<h1>Hello, HTML</h1>`
    - h1 태그를 사용했으므로, "h1 요소"라고 부름
    - *요소: 내용을 포함한 태그 전체, 웹 페이지를 구성하는 기본 단위*

- 태그는 HTML 뿐만 아니라 XML, SGML, XHTML 등 다양한 마크업 언어에서 사용

## 속성 Attribute

- 속성은 HTML 요소에 추가적인 정보를 제공
- 요소의 동작, 스타일, 내용에 영향을 미칠 수 있음
- 속성명과 속성값으로 구성
- **시작태그 안**에 작성, `속성명="속성값"` 의 형식
- 속성값을 나타낼때는 쌍따옴표`"`와 홑따옴표`'` 모두 사용 가능

```
<h1 id="title" class="main">Hello, HTML</h1>
```

- 속성명: `id`, `class`
    - 속성명은 대소문자를 구분하지 않지만, 보통 소문자로 작성
- 속성값: `title`, `main`
- 의미와 용도에 따라 여러 속성이 존재

<br>

- 하나의 태그에 여러 속성을 선언할 수 있음
    - 여러 속성을 선언할 때는 공백으로 구분
    - 속성은 선언 순서에 관계 없이 동일하게 작동

<br>

- 속성의 종류
    - 글로벌 속성: 모든 HTML 요소에 사용 가능
    - 필수 속성: 특정 HTML 요소에서 반드시 설정해야 하는 속성
    - 선택적 속성: 특정 요소에 필요에 따라 추가할 수 있는 속성
    
## 태그의 중첩

- 태그의 중첩이란, HTML 태그 안에 다른 HTML 태그를 포함시키는 구조
- 태그의 중첩을 통해 부모 태그와 자식 태그의 관계 형성
- 부모 태그 안에 자식 태그가 포함된 형태

```
<h1>Hello, <i>HTML</i></h1>
```

- 부모 태그: `<h1>`, `</h1>`
- 자식 태그: `<i>`, `</i>`
- 자식 태그는 부모 태그를 벗어나면 안됨

## 빈 태그 Empty Tag

- 빈 태그 (Empty Tag)는 내용(Content)이 없는 태그를 의미
- 시작 태그와 종료 태그가 하나로 합쳐져 있는 형태
- 그 자체로 속성만을 갖거나 특정 기능을 수행
- HTML5에서는 빈 태그를 종료 태그 없이 작성
    - 예를 들어 `<br>`
- 주요 빈 태그들
    - `<br>`: 줄 바꿈
    ```
    <p>첫 번째 줄<br>두 번째 줄</p>
    ```
    - `<img>`: 이미지 파일을 웹 페이지에 삽입
        - 속성: src(이미지 경로), alt(대체 텍스트) 등
    ```
    <img src="image.jpg" alt="Example Image">
    ```
    - `<hr>`: 수평선 삽입, 구분선
    ```
    <hr>
    ```
    - `<input>`: 사용자가 데이터를 입력할 수 있는 폼 요소 생성
    ```
    <input type="text" placeholder="Enter your name">
    ```
    - `<meta>`: 문서의 메타 정보를 제공, 보통 `<head>`안에 사용
    ```
    <meta charset="UTF-8">
    ```
    - `<link>`: 외부 리소스(ex: 스타일시트 파일)와 연결을 설정
    ```
    <link rel="stylesheet" href="styles.css">
    ```

## 공백

- 일반적으로 HTML은 공백을 여러 개 입력해도 하나로 처리
- 여러 번의 공백 문자, 탭, 줄 바꿈(개행)을 사용하더라도 하나의 공백만 화면에 표시
- 공백 문자
    - 스페이스(Space): 한 번의 공백 문자
    - 탭(Tab): `\t`로 표현
    - 줄 바꿈(Line Break): 텍스트나 코드를 작성할 때 줄을 바꾸면 HTML에서 공백으로 처리되지만, 
                          화면에서 줄 바꿈을 나타내려면 `<br>` 태그를 사용
- `&nbsp;` (Non-Breaking Space)
    - HTML에서 여러 개의 공백을 정확하게 표시하고 싶을 때는 `&nbsp;`를 사용
    - 공백을 의미하며, 연속적으로 여러 번 사용 가능
- `<pre> </pre>` 태그
    - preformatted text의 약자, 서식이 지정된 텍스트
    - 텍스트가 HTML 문서에서 입력된 그대로 출력되도록 함
    - 태그 안의 내용은 공백, 탭, 줄 바꿈 등이 그대로 표시 됨
    - 태그 안에서 텍스트는 고정폭(fixed-width) 글꼴로 서식이 지정되어 있음
    - HTML 문서에서 코드를 표시할 때 자주 사용 됨
        - 코드 내의 들여쓰기와 줄 바꿈을 정확히 유지해야 할 때 유용
    - HTML 문서에서 특별한 서식이 필요한 텍스트를 표시할 때 사용
    - CSS 스타일 적용 가능
        - 고정폭 글꼴을 사용하지만, CSS를 사용하여 글꼴, 텍스트 색상 등의 스타일을 변경 가능

## 주석

- 주석은 코드 내에서 설명을 추가하거나 특정 부분을 숨기고 싶을 때 사용
- 주석은 가독성을 높이고, 수정하거나 유지 보수할 때 유용
- 주석은 웹 페이지에 표시되지 않으며, 오직 소스 코드 안에서만 보임
- 주석은 브라우저에 의해 무시되기 때문에 페이지 렌더링에 영향을 주지 않음
- `<!-- 내용 -->`의 형식

## HTML의 기본 구조

```
<!DOCTYPE html>
<html lang="ko">
    <head>
        <meta charset="UTF-8">
            <title>HTML</title>
    </head>
    <body>
        <h1>Hello, HTML</h1>
    </body>
</html>
```
- `<!DOCTYPE html>`: HTML5 문서임을 브라우저에 알림
    - `DOCTYPE`: 문서 타입 정의
    - `html`: HTML5 임을 나타냄
- `<html>`: HTML 문서의 최상위 요소, 문서의 시작과 끝을 정의
    - `lang="ko"`: 문서의 언어= 한국어
- `<head>`: 문서의 메타데이터를 포함, 주로 문서 제목, 문자 인코딩, 뷰포트 설정, 외부 스타일 시트 파일 및 js 파일 연결 등의 역할
    - `<meta charset="UTF-8">`: 문자 인코딩 방식을 UTF-8로 설정
    - `<title>`: 브라우저 탭에 표시될 문서 제목을 설정
- `<body>`: 실제 브라우저 화면에 나타나는 내용이 들어가는 부분, 텍스트, 이미지, 링크, 버튼 등 사용자가 보는 모든 요소를 포함
    - `<h1>`: 가장 큰 제목
    - `<p>`: 문단을 정의

## 다양한 태그

### 제목과 단락요소

#### 제목 태그 `<h>`
- 헤딩(Heading) 태그라고도 불림
- 문서 내에서 제목을 정의
- h1~h6까지 다양한 레벨이 존재
- 각각은 제목의 중요도나 계층을 나타냄
- `<h1>`이 가장 큰 제목, `<h6>`이 가장 작은 제목
- CSS로 크기 변경 가능
- 검색 엔진 최적화(SEO)에서 중요
    - `<h1>`태그는 페이지의 핵심 주제를 나타내기 때문에 검색 엔진 최적화에 중요한 역할
    - 보통 검색 엔진은 `<h1>`을 통해 페이지의 주요 주제를 이해
- 기본적으로 각 헤딩을 굵은 글씨로 표시, `<h1>`부터 `<h6>`까지 점차적으로 글꼴 크기가 작아짐

#### 단락 태그 `<p>`
- 단락(paragraph)을 정의하는 태그
- 텍스트 콘텐츠를 단락 단위로 구분하고, 가독성을 높이기 위해 사용
- 기본적으로 상하에 여백을 추가하여, 단락 간 구분을 명확히 함
- 블록 요소로써, 페이지에서 새로운 줄에서 시작하며, 기본적으로 100%의 너비를 차지
- CSS로 스타일링 가능

#### 개행 태그 `<br>`
- 줄 바꿈(line break)을 의미하는 태그
- 텍스트나 콘텐츠 내에서 강제로 줄을 바꿀 때 사용
- 인라인 요소로, 다른 텍스트와 같은 라인에 포함되어 줄 바꿈만 일으킴
- 텍스트나 콘텐츠 사이에 위치, 블록 요소처럼 새로운 영역을 만들지 않음
- 여러 번 사용 가능

### 텍스트 표현 태그

#### `<b>`
- 텍스트를 굵은 글씨(Bold)로 표현
- 시각적으로 강조, 단순히 텍스트 스타일을 변경

#### `<i>`
- 텍스트를 기울여서 표시
- Italic의 약자
- 강조, 인용, 책 제목 등 의미적으로 구분하려는 텍스트에 사용
- 단순히 텍스트 스타일을 위한 사용도 가능

#### `<u>`
- 텍스트에 밑줄(Underline)을 추가
- 밑줄은 일반적으로 하이퍼링크를 나타내는 데 사용되므로, 남용하지 않는 것이 좋음

#### `<s>`
- 텍스트에 취소선을 추가
- Strikethrough의 약자
- 수정된 텍스트나 취소된 내용을 표시할 때 사용

### 앵커 태그

#### `<a>`
- Anchor 태그
- 주로 하이퍼링크를 만들 때 사용
- 이 태그를 사용하면 사용자가 클릭할 수 있는 링크를 생성
- 다른 웹 페이지, 웹 페이지 내의 다른 위치, 파일 다운로드, 이메일 전송 등 다양한 용도로 사용
- Anchor 태그의 속성
    - `href`(Hypertext Reference)
        - 앵커 태그의 필수 속성
        - 링크할 주소(URL)를 지정
        - 페이지 간 이동: 같은 도메인 내 다른 HTML 페이지로 링크
        <br>현재 페이지와 동일한 폴더 내에 위치한 파일을 참조
        ```
        <a href="about.html">About Us</a>
        ```
        - 같은 페이지 내 특정 위치로 이동: 앵커 링크(Anchor Link)
        <br>특정 위치에 `id`속성을 부여하고, 그 위치를 링크로 연결
        <br>예를 들어 빠르게 화면 최상단으로 이동하고자 할 때
        ```
        <h2 id="services">Our Services</h2>  <!--목표 위치-->
        <p>Here are the services ...</p>
        ...
        <a href="#services">Go to Services Section</a>  <!--목표 위치로 이동 링크 추가-->
        ```
        - 다른 페이지의 특정 위치로 이동: 외부 페이지의 특정 섹션으로 이동
        <br>다른 페이지 내의 특정 위치로 이동하려면, 그 페이지의 URL 뒤에 `#id`를 추가하여 링크를 생성
        ```
        <a href="about.html#services">Go to Services Section</a>
        <!--about.html 페이지의 id=services인 섹션으로 이동-->
        ```
    - `target`
        - 링크를 열 때의 창이나 탭을 설정, 링크된 리소스를 어디에 표시할지 설정
        - `_self`: 기본값, 현재 창이나 탭에 표시
        - `_blank`: 새로운 탭, 새로운 창에 표시, 외부 페이지가 나타나도록 함
        - `프레임이름`: 특정 프레임에서 열도록 지정
        - `_parent`: 부모 프레임에서 열도록 지정, 부모 프레임이 없으면 `_self`와 같은 동작
        - `_top`: 링크의 최상위 창에서 열도록 지정

### 의미가 없는 컨테이너 태그, 구조적 태그

- 특정한 의미를 가지지 않지만, 스타일링이나 레이아웃을 위한 그룹화를 위해 사용

#### `<div>`
- division의 약자
- 웹 페이지에서 다른 요소들을 그룹화하거나 섹션을 나누는 데 사용
- 주로 스타일링이나 레이아웃을 조정할 때 사용되며, 그 자체로는 의미를 가지지 않음
- 블록 레벨 요소에 속함
    - *블록 레벨 요소: 새로운 줄을 시작, 100% 너비를 차지*

#### `<span>`
- 단어의 약자가 아님, 범위나 부분을 나타내는 의미
- 페이지 내에서 특정 부분을 스타일링하거나 JavaScript로 다룰때 사용
- 인라인 요소에 속함
    - *인라인 요소: 새로운 줄을 시작하지 않고, 기존 요소들과 같은 라인에서 사용, 너비와 높이를 직접 설정할 수 없음*

### 리스트 요소

#### `<li>`

- list item, 목록의 항목을 나타내는 태그

#### `<ul>`

- 순서가 없는 리스트(unordered list)를 나타내는 태그
- CSS를 이용한 스타일링 가능
```
<ul>
    <li>사과</li>
    <li>배</li>
    <li>포도</li>
</ul>
```

#### `<ol>`

- 순서가 있는 리스트(ordered list)를 나타내는 태그
- CSS를 이용한 스타일링 가능
```
<ol>
    <li>첫번째: 물 끓이기</li>
    <li>두번째: 스프 넣기</li>
    <li>새번째: 면 넣기</li>
</ol>
```

#### `<dl>`

- 설명 목록(Description list), 용어와 그에 대한 정의나 설명
- `<dt>`와 `<dd>`로 구성
- `<dt>`: 설명할 용어(Definition Term)
- `<dd>`: 용어데 새한 설명이나 정의(Description Definition)<br>dd는 여러개 중복 가능
- CSS를 이용한 스타일링 가능
```
<dl>
    <dt>HTML</dt>
    <dd>HTML은 웹 페이지를 만들기 위한 마크업언어</dd>
    <dd>CSS를 이용한 스타일링 가능</dd>

    <dt>CSS</dt>
    <dd>CSS는 웹 페이지의 스타일을 정의하는 언어</dd>
</dl>
```

### 이미지 요소

#### `<img>`

- 이미지(image)를 웹 페이지에 삽입할 때 사용하는 태그
- 닫는 태그가 필요없는 빈 태그
- `<img src="이미지 경로" alt="이미지 설명">`
- `src`: source의 약자, 필수 속성, 이미지 파일의 경로를 지정
    - 상대 경로: 현재 HTML 파일의 위치를 기준으로 이미지 파일의 위치를 지정
        - `../`: 상위 폴더를 가리키는 경로
    - 절대 경로: 파일이 저장된 위치를 명확히 지정, 전체 URL, 전체 경로
- `alt`: alternative text의 약자, 필수 속성, 이미지가 로드되지 않거나 시각적으로 접근할 수 없는 사용자를 위한 대체 텍스트, 이미지의 내용에 대한 간단한 설명
- `width`, `height`: 이미지의 너비와 높이를 지정(픽셀px 단위), 선택 속성
    - 선언하지 않으면 원본 크기대로 노출
    - 하나만 선언하면 나머지 속성은 선언한 속성의 크기에 맞춰 자동으로 비율 조정
    - 두 속성 모두 선언 시 이미지의 비율과 무관하게 선언한 크기대로 변경
- `title`: 이미지에 마우스를 올렸을 때 나타나는 설명을 지정
- `loading`: 이미지 로딩 방식을 지정
    - `lazy`: 사용자가 이미지가 보이는 위치에 올 때 이미지를 로드
    - `eager`: 페이지가 로드되자마자 이미지를 로드
- CSS를 이용한 스타일링 가능

#### 이미지 파일 형식

- gif : 제한적인 색 사용, 용량이 적음, 투명 이미지와 애니메이션 이미지 지원
- jpg : 사진이나 복잡한 색상을 가진 이미지에 사용, 높은 압축률, 자연스러운 색상 표현을 지원, 투명과 애니메이션을 지원하지 않음
- png : 이미지 손실이 적음, 투명과 반투명 모두 지원, 투명 배경 필요시 사용

### 테이블 요소

- 데이터를 표 형태로 구조화할 때 사용
- 구성 요소: `<table>`, `<thead>`, `<tbody>`, `<tfoot>`, `<tr>`, `<th>`, `<td>`
- CSS를 이용한 스타일링 가능

#### 테이블 구성 요소
- `<table>`: 테이블의 전체 구조를 정의
- `<caption>`: 테이블의 제목을 정의, 각 테이블에는 하나의 `<caption>`만 사용 가능
- `<thead>`: 테이블의 헤더 부분, 일반적으로 열 제목을 포함, 스타일을 적용하거나 스크롤할 때 고정하는 등의 용도
- `<tbody>`: 테이블의 본문 부분, 실제 데이터가 들어가는 곳
- `<tfoot>`: 테이블의 푸터 부분, 데이터 요약이나 합계, 총합 등을 표시
- `<tr>`: 테이블의 행(row)을 정의, 각 `<tr>`요소는 하나의 데이터 행을 나타냄
- `<th>`: 테이블의 헤더 셀을 정의, 보통 열 제목을 표시, 기본적으로 굵은 글씨와 중앙 정령
- `<td>`: 테이블의 데이터 셀을 정의, 실제 데이터가 표시되는 곳
- `<colgroup>`: 테이블의 열 그룹을 정의, 열과 관련된 스타일이나 속성을 그룹으로 묶어서 일괄적으로 적용할 때 사용, 내부에 여러 개의 `<col>` 요소를 넣어 각 열에 대해 스타일링
- `<col>`: `<colgroup>` 내부에 포함되어 각 열에 스타일을 적용하거나 속성을 설정, 각 `<col>`은 하나의 열을 정의


#### 테이블 관련 속성
- `border`: 테이블에 테두리를 추가
    ```
    <table border="1">
    ```
- `colspan`: 셀을 가로방향(열column방향)으로 병합
- `rowspan`: 셀을 세로방향(행row방향)으로 병합
    ```
    <td colspan="2">열방향으로 병합</td>
    <td rowspan="2">행방량으로 병합</td>
    ```
- `align`: 텍스트나 셀의 정렬을 지정(left, center, right 등)
- `scope`: 주로 `<th>`태그에 사용, 헤더 셀이 어떤 범위의 데이터를 대표하는지 정의
    - `row`, `col`, `rowgroup`, `colgroup`, `tabel`
        ```
        <th scope="row">th가 행 헤더임을 나타냄</th>
        ```

### 폼 요소

- 사용자가 웹 페이지와 상호작용할 수 있도록 하는 부분
- 주로 데이터 입력, 선택, 제출을 위해 사용

#### `<form>`

- 폼을 정의하는 태그, 폼 요소들을 감싸는 태그
- 사용자가 서버에 전송할 데이터를 묶는 컨테이너 역할
```
<form action="/submit" method="POST">
    <!-- 폼 요소들 -->
</form>
```
- `action`: 폼 데이터를 전송할 서버의 URL
- `method`: 폼 데이터를 서버로 전송하는 방법(GET, POST 등)
    - `GET`: 데이터가 전송될 때 주소창에 파라미터 형태로 붙어 데이터가 노출
    - `POST`: 데이터가 전송될 때 데이터가 노출되지 않음

#### `<input>`

- 내용이 없는 빈 태그
- `type` 속성에 따라 여러 종류로 사용

- `type`: 입력 필드의 종류를 결정
    - `text`
        - 사용자가 텍스트를 입력할 수 있는 일반적인 입력 필드
        - 주로 아이디, 이름, 주소, 전화번호 등 단순한 텍스트를 입력할 때 사용
    - `password`
        - 비밀번호와 같은 민감한 데이터를 입력받을 때 사용
        - 입력한 문자가 화면에 표시되지 않음
    - `radio`
        - 여러 옵션 중 하나만 선택할 수 있도록 하는 라디오 버튼을 만들 때 사용
        - 라디오 버튼은 중복 선택이 불가능, 하나의 항목만을 선택
        - checked 속성은 값이 별도로 존재하지 않는 boolean 속성, boolean속성은 true/false 둥 중 하나의 값을 가지며 속성이 존재하면 true, 속성이 존재하지 않으면 false를 가짐
        - name 속성은 라디오 버튼과 체크박스를 그룹화시켜주는 속성
    - `checkbox`
        - 여러 옵션 중 하나 이상을 선택할 수 있는 체크박스를 만들 때 사용
        - 체크박스는 중복 선택이 가능
    - `file`
        - 파일을 서버에 업로드할 수 있는 입력 필드를 생성
        - 브라우저에 따라 표현되는 형태가 다르지만 모두 같은 역할
    - `submit`
        - 폼을 제출하는 버튼을 생성
    - `reset`
        - 폼의 값을 초기화하는 버튼을 생성
    - `image`
        - 이미지가 삽입된 버튼을 생성
        - 이미지 태그 처럼 `src`와 `alt` 속성이 필요하며, width/height 속성 적용 가능
    - `button`
        - 아무 기능이 없는 버튼을 생성
    - `email`
        - 이메일 주소를 입력받는 필드
        - 입력된 값이 이메일 형식인지 검증 가능
        - 이메일 형식이 맞지 않는 경우, 폼 제출 시 브라우저가 자동으로 경고 메세지를 표시하고 사용자가 올바른 형식으로 입력할 때까지 제출을 막음
    - `number`
        - 숫자만 입력받을 수 있는 필드
        - 숫자 범위나 특정 값의 조건을 지정 가능
        - `min`, `max`, `step`
        - 일부 브라우저에서는 유효성 검사를 수행, 잘못된 값을 입력하면, 폼 제출 시 경고 메세지를 표시
    - `tel`
        - 전화번호를 입력받는 필드
        - 입력된 값이 전화번호 형식에 맞는지 검사하지 않지만, 사용자에게 전화번호를 입력하도록 유도
        - 일부 브라우저 확장 프로그램에나 JavaScript를 사용하여 전화번호 형식의 유효성 검사 가능
    - `url`
        - URL을 입력받는 필드
        - 사용자가 입력한 값이 URL 형식인지 검증 가능
        - 브라우저가 사용자가 입력한 값이 URL 형식에 맞는지 유효성 검사 수행
        - 유효하지 않은 URL을 입력하면, 폼 제출 시 경고 메세지 표시
    - `date`
        - 날짜를 입력받는 필드
        - 달력 형식으로 날짜를 선택 가능
    - `time`
        - 시간을 입력받는 필드
        - 시간을 선택하는 UI를 제공
    - `datetime-local`
        - 날짜와 시간을 입력받는 필드
        - 날짜와 시간을 선택하는 UI를 제공
    - `range`
        - 슬라이더로 값을 선택할 수 있는 입력 필드
        - 값의 범위 지정 가능
        - `min`, `max`, `step`, `value`
    - `hidden`
        - 화면에 보이지 않지만, 폼이 제출될 때 데이터를 서버로 전송하는 숨겨진 필드

- `name`
    - 서버로 전송될 입력 필드의 이름을 정의
    - 서버로 전송될 폼 데이터의 키
    - 서버는 키(`name`)와 값(`value`) 쌍을 받아 처리
    - `name`속성이 없으면, 해당 요소의 값은 서버로 전송되지 않음
    - 동일한 `name`속성을 가진 여러 개의 폼 요소가 있을 경우, 서버로 전송되는 값은 배열로 처리 됨
    - 공백 사용 불가
- `placeholder`: 필드에 대한 힌트를 제공하는 텍스트(입력되면 사라짐)
- `value`: 사용자가 폼 요소에 입력하거나 선택한 값, 입력 필드의 기본값을 설정, 버튼에 표시되는 이름 변경
- `id`: 특정 폼 요소를 고유하게 식별하기 위한 값(주로 `<label>`태그의 `for`속성과 연결)
- `required`: 필수 입력 필드임을 지정, 사용자가 데이터를 입력해야만 폼 제출 가능
- `readonly`: 입력 필드를 읽기 전용으로 설정
    - 사용자가 데이터를 보기만 할 수 있도록 하고, 수정은 할 수 없게 함
    - 텍스트를 복사하거나 선택할 수 있음
    - 폼 제출 시 해당 값을 서버로 전송
- `disabled`: 입력 필드를 비활성화 상태로 설정
    - 사용자가 해당 필드를 클릭, 선택, 입력, 수정할 수 없게 함
    - 폼 제출 시 해당 값이 서버로 전송되지 않음

#### `<select>`와 `<option>`

- 드롭다운 목록(선택 목록 상자, 콤보박스)을 생성
- 몇 개의 선택지를 리스트 형태로 노출하고 그중 하나를 선택할 수 있게 하는 태그
- `<select>`내부의 `<option>`으로 각 항목을 나타냄
- `<select>`의 속성
    - `name`
    - `id`
    - `size`: 드롭다운 목록에 표시할 항목의 수를 정의
        - 기본적으로는 드롭다운 형태로 표시되지만,
        이 속성을 사용하면 여러 항목을 한 번에 볼 수 있는 목록으로 변경 가능
    - `multiple`: 여러 항목을 동시에 선택 가능
- `<option>`의 속성
    - `value`: 항목을 선택했을 때 서버로 전송되는 값을 정의
    - `selected`: 기본값으로 설정, 페이지가 로드될 때 이 항목이 기본적으로 선택된 상태로 표시
    - `disabled`: 해당 항목을 비활성화
    - `<optgroup>`으로 옵션 항목을 그룹화 가능
        - `label`: 그룹의 이름 설정
        - CSS를 이용한 스타일링 가능

#### `<textarea>`

- 여러 줄로 입력할 수 있는 텍스트 영역
- 내용: 기본값 설정 가능
- `rows`: 세로 크기를 조절하는 속성 (화면에 보여지는 줄 수)
- `cols`: 가로 크기를 조절하는 속성 (한 줄에 들어가는 글자의 수,
    수치의 의미는 평균적인 글자의 너비로 정확히 글자 수를 나타내지는 않음)
- `placeholder`: 예시 텍스트 표시, 기본 입력값이 없을때만 표시
- `maxlength`: 텍스트 영역에 입력할 수 있는 최대 문자 수
- `wrap`: 텍스트를 입력할 때의 줄 바꿈 양식
    - `soft`: 기본적으로 입력한 텍스트가 줄 바꿈 없이 자동으로 표시
    - `hard`: 사용자가 입력할 때 줄 바꿈이 실제로 적용

#### `<button>`

- 버튼을 만들 때 사용
- `type`
    - `submit`: 폼 데이터를 서버로 전송하는 버튼
    - `reset`: 폼 내의 모든 필드를 초기화하는 버튼
    - `button`: 기본 버튼, 아무 동작도 수행하지 않음, 주로 자바스크립트와 함께 사용
- `name`, `value`
- `disabled`
- `autofocus`: 페이지 로드 시 자동으로 해당 버튼에 포커스가 맞춰짐
- `form`: 어떤 폼에 속하는지 명시
    - `<form>`태그 외부에 배치된 버튼이 특정 폼과 연결되도록 사용 가능
    - `<form>`태그의 `id`속성과 같도록 하여 연결
- 빈 태그가 아니며 내용을 안에 직접 넣을 수 있음
- CSS를 이용한 스타일링 가능

#### `<label>`

- 폼 요소에 대한 설명을 추가
- 입력 필드와 관련된 텍스트를 사용자에게 설명할 때 사용
```
<label for="name">이름</label>: <input type="text" id="name">
```
- 폼 요소의 `id` 속성값과 `<label>`의 `for` 속성값을 같게 적어야 함
- `<label>`을 사용하면 이를 클릭했을 경우 해당 폼 요소를 클릭한 것처럼 동작
- 스크린 리더기를 통해 듣게 되면 해당 폼 요소에 접근 시 `<label>`을 함께 읽어주게 됨

#### `<fieldset>`과 `<legend>`

- 여러 폼 요소들을 그룹화
- `<fieldset>`: 여러 개의 폼 요소를 그룹화하여 구조적으로 만들기 위해 사용
- `<legend>`: 해당 그룹에 대한 제목을 정의, `<fieldset>` 내부에 작성
    - 라틴어 legenda에서 유래, 읽어야할 것, 설명이라는 뜻
- `<fieldset>`과 `<legend>` 모두 CSS를 이용한 스타일링 가능

## 콘텐츠 모델 (Content Models)

- HTML 요소가 허용하는 자식 요소와 그 자식 요소들의 순서를 정의하는 규칙이 존재하는데, 그 규칙에 대해 비슷한 성격의 요소들끼리 그룹화한 것

### Metadata Content

- 콘텐츠의 스타일, 동작을 설정하거나 다른 문서와의 관계 등 정보를 포함하는 요소들이 포함
- 일반적으로 문서의 헤더나 설정 정보에 해당하는 요소들이 속함
- `<base>`, `<link>`, `<meta>`, `<script>`, `<style>`, `<title>` 등

### Flow Content

- 문서의 자연스로운 흐름에 의해 배치되는 요소들
- 문서 내 대부분의 콘텐츠를 포함
- 웹 페이지의 주요 구조를 정의하는데 사용
- `<a>`, `<article>`, `<div>`, `<footer>`, `<header>`, `<p>` 등

### Sectioning Content

- 문서의 구조와 관련된 요소
- 주로 콘텐츠를 구획화하여 더 잘 이해할 수 있도록 만드는 역할
- `<article>`, `<aside>`, `<footer>`, `<header>`, `<nav>`, `<section>` 등

### Heading Content

- 섹션의 헤더를 정의
- 문서의 각 섹션을 식별, 문서 내에서 계층적인 제목을 설정
- `<h1>`, `<h2>`, `<h3>`, `<h4>`, `<h5>`, `<h6>` 등

### Phrasing Content

- 텍스트와 관련된 콘텐츠를 구성하는 요소
- 텍스트 또는 텍스트를 꾸며주는 문단 내부 레벨로 사용되는 요소
- `<a>`, `<abbr>`, `<b>`, `<code>`, `<em>`, `<i>`, `<span>`, `<strong>` 등

### Embedded Content

- 외부 콘텐츠를 표현하는 요소
- 이미지, 비디오, 오디오 등 멀티미디어 관련 요소
- `<audio>`, `<canvas>`, `<embed>`, `<iframe>`, `<img>`, `<math>`, `<object>`, `<svg>`, `<video>` 등

### Interactive Content

- 사용자와 상호작용으르 하는 요소
- 대표적으로 폼 요소들
- `<a>`, `<audio>`, `<button>`, `<details>`, `<input>`, `<label>`, `<select>`, `<textarea>` 등

## 시멘틱 마크업 (Semantic Markup)

- HTML에서 의미를 담은 태그를 사용하여 문서 구조를 정의하는 방식
- 내용의 목적과 관계를 정의하여 문서가 더 명확하고 이해하기 쉬운 구조를 갖도록 함
- 기계가 잘 이해할 수 있도록 함
- 웹페이지의 구조와 의도가 명확해지기 때문에 검색 엔진 최적화(SEO)와 접근성(Accessibility) 측면에서 매우 유리
- 의미에 맞는 태그, 요소를 사용
- 시멘틱 마크업은 그 자체로 무엇을 나타내는지 알려주므로, 코드의 가독성이 높아짐
    ```
    <b>굵은</b> vs <strong>중요한</strong>
    <i>기울어진</i> vs <em>강조하는</em>
    <u>밑줄친</u> vs <ins>새롭게 추가된</ins>
    <s>중간선이 있는</s> vs <del>삭제된</del>
    ```
- `<b>`는 의미 없이 단순히 굵게 vs `<strong>`은 중요하다는 의미
- `<i>`는 의미 없이 기울어진 글자 vs `<em>`은 글자의 특정 부분을 강조하는 의미 (emphasis)
- `<u>`와 `<s>`는 의미 없이 글자에 선을 그음 vs `<ins>`와 `<del>`은 내용이 새롭게 추가되거나 삭제 되었다는 의미 (inserted, deleted)

### HTML5에서 새로 생긴 시멘틱 요소

#### `<article>`

- 독립적이고 독자적인 콘텐츠 블록
- 보통 블로그 포스트, 뉴스 기사, 포럼 글 등
- 기존에는 주로 `<div>`나 `<section>` 태그로 감쌌음

#### `<aside>`

- 문서의 본문과 간접적으로 연관된 부가적인 콘텐츠
- 사이드바, 참고자료, 인용문 등
- 기존에는 `<div>`로 감쌌음

#### `<figcaption>`

- `<figure>`태그 내부에 그림, 사진, 다이어그램, 차트 등의 설명을 추가
- 기존에는 `<p>`나 `<div>`로 감쌌음

#### `<figure>`

- 독립적인 콘텐츠
- 이미지, 비디오, 차트, 다이어그램 등
- `<figcaption>`과 함께 사용
- 기존에는 `<div>`로 감쌌음

#### `<footer>`

- 문서나 섹션의 바닥글
- 저작권 정보, 연락처, 관련 링크 등
- 기존에는 `<div>`로 감쌌음

#### `<header>`

- 문서나 섹션의 머리말
- 사이트 로고, 네비게이션 메뉴, 제목 등
- 기존에는 `<div>`로 감쌌음

#### `<main>`

- 페이지의 주요 콘텐츠
- 문서에서 한 번만 사용되어야 하며, 주로 페이지의 핵심 내용이 담김
- 기존에는 `<div>`로 감쌌음

#### `<mark>`

- 강조할 텍스트에 사용
- 검색어 하이라이트나 중요한 단어 등을 나타낼 때 사용
- 기존에는 `<strong>`이나 `<em>`으로 감쌌음

#### `<nav>`

- 웹사이트의 내비게이션 링크를 묶는 태그
- 문서의 부분 중 현재 페이지내, 또는 다른 페이지의 링크를 보여주는 구획을 나타냄
- 보통 메뉴나 페이지 이동을 위한 링크를 포함
- 기존에는 `<div>`나, `<ul>`로 감쌌음

#### `<section>`

- 문서 내에서 하나의 주제나 섹션을 구분하는 태그
- 더 적합한 의미를 가진 요소가 없을 때 사용
- 여러 콘텐츠들이 관련이 있을 때 이를 묶는 데 사용
- 기존에는 `<div>`로 감쌌음

#### `<time>`

- 날짜와 시간 정보를 나타냄
- 정확한 날짜나 시간 형식을 포함할 수 있음
- 기존에는 `<div>`나 `<span>`으로 감쌌음

#### `<form>`

- 사용자 입력을 받는 양식을 정의
- `<input>`, `<select>`, `<textarea>`, `<button>`등의 태그와 결합
- 기존에도 `<form>`을 사용했지만 다양한 속성들이 추가됨
- 새로운 타입: `email`, `url`, `tel`, `number`, `date`, `time`, `range`등
- 폼 검증 기능: `required`, `pattern`, `min`, `max`
- 자동 완성 기능: `autocomplete="on"`

#### `<details>`

- 추가적인 세부 정보를 사용자에게 제공
- 기본적으로 접힌 상태로 표시, 클릭하면 세부 내용이 펼쳐짐
- 기존에는 JavaScript와 CSS로 기능을 구현했음

#### `<summary>`

- `<details>`요소 내에서 사용
- 사용자에게 표시되는 제목 역할
- 사용자가 클릭할 수 있는 요약 텍스트를 제공
- 사용자가 `<summary>`의 내용을 클릭하면, 그 아래에 있는 `<details>`의 내용이 나타남
- 기존에는 JavaScript와 CSS로 기능을 구현했음

## 블록 레벨 요소와 인라인 레벨 요소

### 블록 레벨 요소

- 새로운 줄에서 시작, 가로 전체를 차지하는 요소
- 블록처럼 페이지에서 차지하는 공간을 확보, 그 아래로 다른 요소가 배치
- 내부에 인라인 요소들을 포함할 수 있음
- `width`, `height` 등의 크기 속성을 설정 가능
- `margin`, `padding` 등을 사용해 공간 조정 가능
- `<article>`, `<footer>`,`<form>`, `<div>`, `<h1>`~`<h6>`, `<header>`, `<ol>`, `<p>`, `<section>`, `<table>`, `<ul>` 등

### 인라인 레벨 요소
- 내용의 흐름에 따라 한 줄 안에 다른 요소들과 나란히 배치되는 요소
- 새로운 줄을 생성하지 않으며, 다른 인라인 요소들과 나란히 배치
- 블록 레벨 요소 안에 포함될 수 있음
- `width`, `height` 등의 크기 속성을 지정 불가능
- `margin`, `padding` 등을 사용해 공간 조정은 가능하지만, 크기 조정은 어려움
- 텍스트나 내부 콘텐츠의 크기에 맞춰 크기가 결정 됨
- 예외로 HTML5 버전에서 `<a>`는 인라인 레벨 요소지만, 자식으로 블록 레벨 요소를 가질 수 있음
- `<a>`, `<code>`, `<em>`, `<i>`, `<img>`, `<span>`, `<strong>` 등
